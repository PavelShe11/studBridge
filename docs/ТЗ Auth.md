# ТЗ Auth — Микросервис

## Введение

**Цель проекта:** Разработать микросервис аутентификации и авторизации пользователей, обеспечивающий безопасную регистрацию, вход, управление сессиями и правами доступа.

**Область применения:** Микросервис предназначен для интеграции с веб, desktop и мобильными приложениями, использующими распределенную архитектуру.

## Требования к системе

### Функциональные требования

- Регистрация пользователей (Фамилия, Имя, email)
- Аутентификация (одноразовым кодам на email)
- Управление сессиями (refreshToken, accessToken)
- Логирование действий пользователей *(Опционально)*

### Нефункциональные требования

- Масштабируемость (горизонтальное масштабирование)
- Безопасность (шифрование данных, защита от атак)
- Совместимость с REST API (для клиент-серверного взаимодействия) и gRPC (для микросервисного взаимодействия)

## Архитектура системы

Основные компоненты:
- **API Gateway** — маршрутизация, rate limiting *(опционально)*
- **Auth Service** — основная логика аутентификации и авторизации
- **PostgreSQL** — хранение данных пользователей, связанных с аутентификацией + роли

## Описание процессов

### Регистрация

На стороне auth не храним пользователей — они хранятся на стороне networking.

- Создание или продление сессии регистрации
- Подтверждение регистрации кодом

> В процессе разработки не будет электронной почты. Код подтверждения пишется в ответе `POST /v1/registration`
> В БД сохраняем только email, код и дату, когда истекает код

## REST API

### Регистрация

#### Создание или продление сессии регистрации (`POST /auth/v1/registration`)

> **Пока всё связанное с nickname закомментировать в коде. Не MVP**

**Требования к nickname:**
- Уникальность
- Состоит только из латинских букв (верхний и нижний регистры) и цифр (другие символы запрещены)

**Возможные ошибки:**
- Пустое поле (относится ко всем полям)
- Такой nickname уже существует
- В nickname используются запрещённые символы
- Введённая строка не является электронной почтой
- Учебное заведение с доменом [domain] не зарегистрировано в Communicator
- Не принято пользовательское соглашение
- Общие ошибки

**Ошибки, которые пользователь НЕ должен получить:**
- Данный email уже зарегистрирован *(не возвращаем с целью сохранения приватности)*

**Входные данные:**
```json
{
  "email": "peterpetrov@edu.ru",
  "acceptedPrivacyPolicy": true,
  "acceptedPersonalDataProcessing": true
}
```

**Выходные данные — success:**
```json
{
  "codeExpires": 1740559007,
  "code": "12345"
}
```

**Выходные данные — error (общая):**
```json
{
  "error": "Сервер не отвечает"
}
```

**Выходные данные — error (400 Bad Request):**
```json
{
  "error": "Ошибка регистрации",
  "detailedErrors": [
    { "field": "firstName", "message": "Поле пустое" },
    { "field": "lastname", "message": "Поле пустое" },
    { "field": "email", "message": "Поле пустое" },
    { "field": "acceptedPrivacyPolicy", "message": "Не принято пользовательское соглашение" },
    { "field": "acceptedPersonalDataProcessing", "message": "Не принято пользовательское соглашение" }
  ]
}
```

---

#### Подтверждение почты (`POST /auth/v1/registration/confirmEmail`)

**Возможные ошибки:**
- Неверный код подтверждения (`invalid_verification_code`)
- Код подтверждения истёк (`code_verification_expired`)
- Общие ошибки

**Ошибки, которые пользователь НЕ должен получить:**
- `registrationSessionId` не существует *(всегда возвращаем 200)*

**Входные данные:**
```json
{
  "email": "peterpetrov@edu.ru",
  "acceptedPrivacyPolicy": true,
  "acceptedPersonalDataProcessing": true,
  "code": "12345"
}
```

**Выходные данные — success (200)**

**Выходные данные — error:**
```json
{
  "error": "Неверный код подтверждения",
  "codeError": "invalid_verification_code"
}
```

---

### Вход

#### Отправка кода подтверждения на почту (`POST /auth/v1/login/sendCodeEmail`)

**Возможные ошибки:** Общие ошибки

**Ошибки, которые пользователь НЕ должен получить:**
- Пользователь не существует или email не зарегистрирован *(возвращаем `codeExpires` и `code`)*

**Входные данные:**
```json
{
  "email": "example.email@ystu.ru"
}
```

**Выходные данные — success:**
```json
{
  "codeExpires": 1740559007,
  "code": "12345"
}
```

**Выходные данные — error:**
```json
{
  "error": "Сервер не отвечает"
}
```

---

#### Подтверждение почты при помощи кода (`POST /auth/v1/login/confirmEmail`)

Использовать запрос `/networking/v1/private/account/validation` для проверки на наличие пользователя.

**Возможные ошибки:**
- Неверный код подтверждения (`invalidVerificationCode`)
- Код подтверждения истёк (`code_verification_expired`)
- Общие ошибки

**Ошибки, которые пользователь НЕ должен получить:**
- email не существует *(всегда возвращаем 400 «Неверный код подтверждения»)*

**Требования к JWT:**
> Использовать в токене только служебную информацию (email, nickname убрать — не несёт смысловой нагрузки)

**Входные данные:**
```json
{
  "email": "example.email@email.ru",
  "code": "12345"
}
```

**Выходные данные — success:**
```json
{
  "accessToken": "<jwt>",
  "accessTokenExpires": 1740559007,
  "refreshToken": "<jwt>",
  "refreshTokenExpires": 1740559007
}
```

**Выходные данные — error (400):**
```json
{
  "error": "Неверный код подтверждения"
}
```

---

## Запросы к Networking (gRPC)

### Проверка данных пользователя (`GET /networking/v1/private/account/validation`)

Валидирует только само значение, не проверяет на обязательность поля.

**Возможные ошибки:**
- Пустое поле (относится ко всем полям)
- Такой nickname уже существует
- Введённая строка не является электронной почтой
- Учебное заведение с доменом [domain] не зарегистрировано в Communicator
- Общие ошибки

**Входные данные:**
```json
{
  "firstName": "Peter",
  "lastName": "Petrov",
  "email": "peterpetrov@edu.ru",
  "acceptedPrivacyPolicy": false,
  "acceptedPersonalDataProcessing": false
}
```

**Выходные данные — success (200):**
```json
{
  "idAccount": "36101ae5-bec7-4adf-a0e7-e147ce68de17"
}
```

**Выходные данные — error (500):**
```json
{
  "error": "Сервер не отвечает"
}
```

**Выходные данные — error (404 / 400):**
```json
{
  "error": "Ошибка регистрации",
  "detailedErrors": [
    { "field": "email", "message": "Поле пустое" },
    { "field": "acceptedPrivacyPolicy", "message": "Не принято пользовательское соглашение" },
    { "field": "acceptedPersonalDataProcessing", "message": "Не принято пользовательское соглашение" }
  ]
}
```

## Безопасность

- JWT с цифровой подписью: `accessToken` и `refreshToken`
- В JWT сохраняется только служебная информация (username и email не сохраняются)
- Аудит логов безопасности *(Опционально)*
